Installation

Perl modules

First, install the following Perl modules, which are available from CPAN:

File::Temp http://search.cpan.org/~dagolden/File-Temp-0.2301/
File::Slurp http://search.cpan.org/~uri/File-Slurp-9999.19/

For instructions on how to install Per modules, see http://www.cpan.org/modules/INSTALL.html.

Libraries

Before you start compiling eFindSite, make sure that the following libraries are available on your system:

zlib http://www.zlib.net/
gzstream http://www.cs.unc.edu/Research/compgeom/gzstream/
f2c http://www.netlib.org/f2c/
libsvm http://www.csie.ntu.edu.tw/~cjlin/libsvm/
apcluster 32bit /64bit http://www.psi.toronto.edu/index.php?q=affinity%20propagation

If you have them installed in locations not recognized by your C++ compiler, depending on your configuration, you may want to copy following files to efindsite‚Äê1.0/lib/:

zlib zlib.h, zconf.h, libz.a
gzstream gzstream.h, libgzstream.a
f2c f2c.h, libf2c.a
libsvm svm.h, libsvm.a
apcluster 32bit / 64 bit apcluster.h apclusterunix64.so/apclusterunix32.so

A note on libsvm: By default, make does not create libsvm.a. Use the archiver ar as shown below to get the required file:

[src]$ tar xzf libsvm-3.17.tar.gz

[src]$ cd libsvm-3.17/

[libsvm-3.17]$ make

[libsvm-3.17]$ ar cr libsvm.a svm.o

Compilers

You need a C++ compiler to compile eFindSite source codes. The GNU compiler (g++) is set as a default compiler in Makefile; you can check which version is installed on your system using -v  option:

[local]$ g++ -v

gcc version 4.4.6 20120305 (Red Hat 4.4.6-4) (GCC)

Installation

Download eFindSite from here, unpack the tarball, go to efindsite-1.0/src/ and run make:

[local]$ tar xzf efindsite-1.0.tar.gz

[local]$ cd efindsite-1.0/src/

[src]$ make

If there are no problems, the following files will show up in efindsite-1.0/bin/:

[src]$ ls ../bin/

efindsite
efindsite_map
efindsite_scr
